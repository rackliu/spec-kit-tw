---
name: translation-execute
description: "智慧執行檔案翻譯, 確保高品質本地化"
---

使用者輸入: 
$ARGUMENTS

目標: 基於檢測結果, 智慧執行高品質的檔案翻譯, 確保語義準確性和本地化品質.

執行步驟: 

1. 翻譯準備階段
   - 載入翻譯標準和術語表
   - 初始化翻譯上下文和記憶
   - 驗證原版檔案完整性

2. 智慧翻譯執行
   - 按優先順序處理翻譯任務: 
     * 核心模板檔案(templates/核心模板)
     * CLI相關檔案(src/specify_cli/中的文案)
     * 專案文件(docs/, README.md等)
     * 輔助檔案(memory/, templates/commands/)
   - 對每個檔案執行: 
     * 內容分析和結構識別
     * 上下文感知的智慧翻譯
     * 術語一致性檢查
     * 格式和連結保持
     * 程式碼塊和佔位符保護

3. 翻譯品質控制
   - 語義準確性驗證
   - 術語一致性檢查
   - 格式完整性驗證
   - 連結和引用有效性檢查
   - 中文表達自然度評估

4. 特殊處理規則
   - CLI命令: specify → specify-tw
   - 路徑引用: 保持原版路徑不變
   - 程式碼塊: 完全保持原樣
   - 佔位符: 保持格式不變
   - 技術術語: 參考術語表

5. 翻譯結果驗證
   - 對比原版確保內容完整性
   - 檢查翻譯覆蓋率和品質
   - 驗證特殊規則執行情況
   - 生成翻譯報告

行為規則: 
- 嚴格遵循 @TRANSLATION_STANDARDS.md 翻譯標準
- 使用 @TERMINOLOGY.md 確保術語一致性
- 採用上下文感知的翻譯策略, 避免孤立翻譯
- 保護所有技術元素(程式碼、連結、佔位符)
- 確保翻譯後的功能與原版完全一致
- 對不確定的翻譯標記為需要人工稽核
- 維護翻譯記憶, 提高後續翻譯品質
