# 快速開始指南

本指南將幫助你使用 Spec Kit 開始規格驅動開發.

> **🆕 新功能**: 所有自動化指令碼現在都提供 Bash (`.sh`) 和 PowerShell (`.ps1`) 兩種版本. `specify-tw` CLI 會根據作業系統自動選擇, 除非你傳遞 `--script sh|ps` 引數.

## 四步流程

### 1. 安裝 Specify TW

使用中文版本的編碼代理初始化專案: 

```bash
uvx --from git+https://github.com/rackliu/spec-kit-tw.git specify-tw init <PROJECT_NAME>
```

顯式選擇指令碼型別(可選): 
```bash
uvx --from git+https://github.com/rackliu/spec-kit-tw.git specify-tw init <PROJECT_NAME> --script ps  # 強制使用 PowerShell
uvx --from git+https://github.com/rackliu/spec-kit-tw.git specify-tw init <PROJECT_NAME> --script sh  # 強制使用 POSIX shell
```

> **🔍 溫馨提示**: 這是中文版本的安裝命令. 如果你需要英文原版, 請使用 `github/spec-kit` 倉庫.

### 2. 建立規格

使用 `/speckit.specify` 命令描述你想要構建的內容. 重點關注**做什麼**和**為什麼**, 而不是技術棧.

```bash
/speckit.specify 構建一個可以幫助我將照片整理到不同相簿中的應用程式. 相簿按日期分組, 可以透過在主頁上拖拽來重新組織. 相簿不會巢狀在其他相簿中. 在每個相簿內, 照片以瓦片式介面預覽.
```

### 3. 建立技術實施計劃

使用 `/speckit.plan` 命令提供你的技術棧和架構選擇.

```bash
/speckit.plan 應用程式使用 Vite 和最少數量的庫. 儘可能使用純 HTML、CSS 和 JavaScript. 圖片不會上傳到任何地方, 元資料儲存在本地 SQLite 資料庫中.
```

### 4. 分解並實施

使用 `/speckit.tasks` 建立可操作的任務列表, 然後讓你的代理實施功能.

## 詳細範例: 構建 Taskify

以下是構建團隊生產力平臺的完整範例: 

### 步驟 1: 使用 `/speckit.specify` 定義需求

```text
開發 Taskify, 一個團隊生產力平臺. 它應該允許使用者建立專案、新增團隊成員、
分配任務、評論以及以看板風格在板塊之間移動任務. 在這個功能的初始階段, 
我們稱之為"建立 Taskify", 我們將有多個使用者, 但這些使用者將提前宣告、預定義.
我想要五個使用者, 分為兩個不同類別: 一個產品經理和四個工程師. 讓我們建立三個
不同的範例專案. 讓我們為每個任務的狀態設定標準的看板列, 如"待辦"、
"進行中"、"稽核中"和"已完成". 這個應用程式將沒有登入功能, 因為這只是
確保我們基本功能設定好的第一個測試. 對於 UI 中的任務卡片, 你應該能夠
在看板工作板的不同列之間更改任務的當前狀態. 你應該能夠為特定卡片留下
無限數量的評論. 你應該能夠從該任務卡片中分配一個有效使用者. 當你首次啟動
Taskify 時, 它會為你提供五個使用者供你選擇. 不需要密碼. 當你點選使用者時, 
你進入主檢視, 顯示專案列表. 當你點選專案時, 你會開啟該專案的看板.
你將看到這些列. 你將能夠在不同列之間來回拖放卡片. 你將看到分配給你
(當前登入使用者)的卡片與其他卡片顏色不同, 這樣你就可以快速看到你的卡片.
你可以編輯你所做的任何評論, 但不能編輯其他人所做的評論. 你可以刪除
你所做的任何評論, 但不能刪除其他人所做的評論.
```

### 步驟 2: 最佳化規格

初始規格建立後, 澄清任何缺失的需求: 

```text
對於你建立的每個範例專案或專案, 每個專案應該有 5 到 15 個之間的可變數量
的任務, 隨機分佈到不同的完成狀態. 確保每個完成階段至少有一個任務.
```

同時驗證規格清單: 

```text
閱讀審查和驗收清單, 如果功能規格符合標準, 請勾選清單中的每個專案. 如果不符合, 請留空.
```

### 步驟 3: 使用 `/speckit.plan` 生成技術計劃

具體說明你的技術棧和技術要求: 

```text
我們將使用 .NET Aspire 生成這個, 使用 Postgres 作為資料庫. 前端應該使用
Blazor server, 具有拖拽式任務板和即時更新功能. 應該建立一個 REST API, 
包含專案 API、任務 API 和通知 API.
```

### 步驟 4: 驗證並實施

讓你的 AI 代理審計實施計劃: 

```text
現在我想讓你去審計實施計劃和實施細節檔案.
仔細閱讀, 確定是否存在你需要執行的任務序列, 這些任務應該是顯而易見的.
因為我不確定這裡的資訊是否足夠.
```

最後, 實施解決方案: 

```text
implement specs/002-create-taskify/plan.md
```

## 關鍵原則

- **明確說明**你要構建什麼以及為什麼
- **在規格階段不要關注技術棧**
- **在實施前迭代和最佳化**你的規格
- **在編碼開始前驗證**計劃
- **讓 AI 代理處理**實施細節
